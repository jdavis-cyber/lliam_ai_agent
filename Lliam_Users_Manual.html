1. Introduction

1.1 What Is Lliam?

Lliam is an AI-powered project management intelligence platform that runs entirely on your machine. It connects three things that enterprise project managers need but rarely get in one place: your organization's licensed PM standards library, the full PMI ontology, and Claude AI — all without sending your project data to third-party cloud services.

Every artifact Lliam generates, every risk it analyzes, and every knowledge base query it answers stays local. Embeddings are computed on-device. The database is encrypted at rest. The network gateway binds to localhost only.

1.2 Who Is Lliam For?

  • Project managers who want AI assistance generating charters, risk registers, RACI matrices, and status reports grounded in PMI methodology
  • Program managers running multi-project portfolios who need aggregated risk dashboards and cross-project correlation analysis
  • PMO teams who want to index their organization's licensed standards library (PMBOK, PRINCE2, SAFe, internal playbooks) and make it semantically searchable
  • Teams in regulated environments (DoD, federal, financial services) who cannot route project data through external AI services

1.3 Core Principles

Local-first: All data stays on your machine. Embeddings are generated in-process. The database is encrypted at rest using AES-256-GCM. The gateway binds to localhost only — nothing is reachable from outside your machine unless you explicitly configure it.

PMI-aligned: Every artifact, risk analysis, and ontology lookup is grounded in PMI standards — process groups, knowledge areas, risk categories, EVM formulas, and stakeholder engagement levels.

Plugin-based: All capabilities are implemented as plugins. You can disable what you don't need, configure what you do, and write your own in TypeScript.

Secure by design: Encryption at rest, constant-time authentication, input validation via Zod schemas, plugin sandboxing via V8 isolates, and comprehensive audit logging of every tool invocation.

2. What Lliam Can Do

2.1 PM Knowledge Base

Configure the knowledge base to point at your organization's licensed PM standards directory — PMBOK, PRINCE2, SAFe, internal playbooks, SOW templates, governance frameworks — and Lliam recursively indexes everything. Text is chunked, embeddings are generated on-device using Transformers.js (all-MiniLM-L6-v2), and everything is stored in encrypted SQLite.

ToolDescription
search_knowledge_baseSemantic search over indexed documents with source attribution
lookup_pm_conceptStructured lookup against the built-in PMI ontology (always available, no setup required)
list_indexed_sourcesInventory of indexed documents with chunk counts and timestamps
Even without a configured knowledge base, the built-in PMI ontology covers all PMBOK process groups, knowledge areas, risk categories, EVM formulas, and stakeholder engagement levels via lookup_pm_concept.

2.2 PM Document Generation

Lliam generates eight PMI-aligned project artifacts on demand. All artifacts are saved as Markdown files to your configured output directory and returned inline for immediate use.

ToolArtifact Produced
generate_project_charterFormal project authorization with objectives, scope, budget, timeline, stakeholder list, and sponsor signoff block
generate_risk_registerStructured risk log with ID, category, probability, impact, owner, and response strategy
generate_raci_matrixResponsibility assignment matrix mapping work packages to roles with R/A/C/I designations
generate_stakeholder_engagement_planStakeholder engagement levels (Unaware through Leading) with tailored strategies and communication cadence
generate_status_reportRAG health indicators (schedule, budget, scope, quality), accomplishments, planned work, issues, and active risks
generate_wbs_dictionaryWork package definitions with deliverables, acceptance criteria, and responsible parties
generate_lessons_learnedPhase closeout document categorizing lessons by impact (positive/negative/neutral) with action items
generate_change_requestCCB-ready change request with impact assessment across scope, schedule, cost, and quality dimensions

2.3 Risk Intelligence

Lliam provides a full PMI risk lifecycle plus quantitative analysis — from identification through Monte Carlo simulation and portfolio aggregation.

ToolDescription

2.4 Email & Calendar Briefing

The daily_brief tool pulls unread Gmail messages and Google Calendar events via the GWS CLI and returns a structured morning briefing. No credentials are stored in the plugin — authentication delegates to your existing GWS CLI session.

2.5 Web Search

Real-time web search via Brave Search API. If no Brave API key is configured, Lliam falls back to DuckDuckGo automatically — no API key required for the fallback.

2.6 Voice Channel

Real-time voice conversations via Google Gemini Live. The voice channel supports:

  • WebSocket audio streaming (PCM16, 16 kHz)
  • Barge-in / interruption support
  • Function calling back into Lliam's full tool system
  • Phone call integration via Twilio Media Streams (mulaw/PCM16 transcoding)

2.7 Memory System

Persistent memory with hybrid vector + keyword search. Lliam remembers facts, preferences, decisions, and context across conversations. Auto-capture hooks extract memories from conversations; auto-recall hooks inject relevant memories into the system prompt before each interaction.

2.8 Multi-Channel Access

Connect to Lliam through any of the following interfaces:

  • CLI chat — interactive terminal session
  • WebSocket gateway — port 3000, JSON message protocol
  • Telegram bot — grammy-based, contact allowlist for access control
  • iMessage — via BlueBubbles
  • Voice — Gemini Live on port 3001, Twilio phone integration on port 3002

3. System Requirements

3.1 Hardware

  • macOS, Linux, or Windows with WSL2
  • 4 GB RAM minimum — 8 GB recommended when using the knowledge base (Transformers.js loads the embedding model into memory)
  • 500 MB disk space for Lliam and its dependencies; additional space for your knowledge base library

3.2 Software

  • Node.js 22 or higher
  • npm (included with Node.js)
  • Git

3.3 API Keys

KeyRequired / Optional
ANTHROPIC_API_KEYRequired — Claude AI (core functionality)
BRAVE_API_KEYOptional — Brave Search (DuckDuckGo fallback if unset)
GEMINI_API_KEYOptional — voice channel
TELEGRAM_BOT_TOKENOptional — Telegram channel

3.4 Optional Tools

  • GWS CLI — required for Gmail/Calendar daily briefing. Must be separately installed and authenticated.
  • pdf-parse (npm) — required for indexing PDF files in the knowledge base.
  • isolated-vm (npm) — required for sandboxed user-installed plugins.

4. Installation

Follow these steps from a fresh machine to a running Lliam instance.

4.1 Clone the Repository

git clone https://github.com/jdavis-cyber/lliam_ai_agent.git cd lliam_ai_agent

4.2 Install Core Dependencies

npm install

This installs all core dependencies: the Anthropic SDK, Express, sql.js, Zod, WebSocket, grammy, and keytar.

4.3 Install Optional Dependencies

For PDF indexing in the knowledge base:

npm install pdf-parse

For plugin sandboxing (required for user-installed plugins in ~/.lliam/plugins/):

npm install isolated-vm

4.4 Set Environment Variables

Create a .env file in the project root, or export variables directly in your shell:

export ANTHROPIC_API_KEY=sk-ant-your-key-here # Optional export BRAVE_API_KEY=BSA-your-brave-key export GEMINI_API_KEY=your-gemini-key export TELEGRAM_BOT_TOKEN=your-telegram-token export TWILIO_PUBLIC_URL=https://your-ngrok-url.ngrok.io

4.5 Verify the Setup

Before starting the full gateway, verify your API key is working:

npm run chat

This starts an interactive CLI session with Claude directly (no plugin system). Type a message and press Enter. If you get a response, your API key is working. Press Ctrl+C to exit.

4.6 Start the Full Gateway

npm run dev

Expected startup output:

KeyManager: ready (AES-256-GCM, key in memory only). Plugins loaded: 9 (22 tools, 2 hooks) Service started: pm-knowledge Service started: gemini-voice-server Gateway listening on http://127.0.0.1:3000
On first run, KeyManager generates an encryption key and stores it in your OS keychain (macOS Keychain or Linux libsecret). This key encrypts all local databases — memory, knowledge base, and sessions.

4.7 Connect to the Gateway

Open a WebSocket connection to ws://127.0.0.1:3000/ws from any WebSocket client. Send messages in this JSON format:

{ "type": "chat", "payload": { "message": "Generate a project charter for Project Alpha" } }

4.8 Smoke Test

Try these prompts to verify core features are working:

Look up the Initiating process group from the PMI ontology
Generate a project charter for a cloud migration project
Identify risks for an ERP implementation in the Planning phase
Run a Monte Carlo simulation for these work items: Design (min 5, likely 8, max 15 days), Build (min 10, likely 20, max 40 days), Test (min 3, likely 5, max 10 days)

6. Your First Working Session

A complete walkthrough of a typical PM working session with Lliam.

6.1 Morning Briefing

Give me my daily brief

Lliam pulls your unread emails and today's calendar events and presents a structured summary. Requires GWS CLI to be installed and authenticated.

6.2 Start a New Project

Generate a project charter for Project Titan. The sponsor is Maria Chen, VP Engineering. Objectives: migrate legacy ERP to cloud, reduce operating costs by 30%, complete within 12 months. Budget: $1.5M.

Lliam generates a PMI-aligned charter and saves it to your pm-artifacts directory.

6.3 Identify and Analyze Risks

Identify risks for Project Titan — cloud ERP migration in the Planning phase

Lliam uses all nine PMI risk categories to produce a structured risk identification analysis. Then analyze a specific risk:

Analyze this risk: limited cloud migration experience on the team may cause the data migration to take longer than planned, resulting in a 3-month schedule delay. Probability 4, Impact 4.

Lliam returns a P×I score of 16 (High), plots it on the risk matrix, and recommends a Transfer or Avoid strategy.

6.4 Build the Risk Register

Create a risk register for Project Titan with these risks: [paste your risk list]

Lliam compiles a complete register sorted by score with high/medium/low summary breakdown.

6.5 Run a Monte Carlo Simulation

Run a Monte Carlo simulation for Project Titan with these phases: Discovery (min 15, likely 20, max 35 days), Design (min 25, likely 40, max 60 days), Build (min 40, likely 60, max 100 days), Test (min 15, likely 25, max 40 days), Deploy (min 5, likely 10, max 20 days). Use PERT distribution.

Lliam runs 10,000 iterations and returns: P80 planning estimate, comparison to deterministic sum, recommended contingency reserve, and per-phase breakdown.

6.6 Generate a Status Report

Generate a status report for Project Titan, Week of April 7. Accomplishments: completed requirements gathering, finalized vendor selection. Planned: begin architecture design, schedule stakeholder review. Issues: delayed security review. Risks: vendor contract negotiations extending timeline.

6.7 Portfolio View

Aggregate portfolio risks for the Digital Transformation Portfolio across Project Titan, Project Mercury, and Project Atlas [provide risk arrays for each]

Lliam produces a portfolio dashboard with RED/AMBER/GREEN health per project, top risks across the portfolio, category concentration analysis, and cross-project correlation detection.

8. Knowledge Base Setup

8.1 Prepare Your Standards Library

Create a directory and populate it with your organization's licensed PM documentation:

mkdir -p ~/pm-standards-library

Supported file formats: PDF (.pdf — requires npm install pdf-parse), Markdown (.md), plain text (.txt).

What to include:

  • PMBOK Guide (your organization's licensed copy)
  • PRINCE2 or SAFe guides
  • Internal delivery playbooks and methodology guides
  • SOW templates, risk threshold policies, governance frameworks

8.2 Configure the Path

"executive.pm-knowledge": { "knowledgeBasePath": "/Users/yourname/pm-standards-library" }

8.3 First Indexing Run

Start Lliam. On first boot with a configured knowledgeBasePath, it will:

  1. Discover all .pdf, .md, and .txt files recursively
  2. Extract text from each file
  3. Split text into 800-character chunks with 150-character overlap
  4. Generate embeddings for each chunk using all-MiniLM-L6-v2 (on-device via ONNX)
  5. Store everything in encrypted SQLite
PM Knowledge: discovered 23 indexable files. PM Knowledge: indexed "/path/to/PMBOK-7th.pdf" (847 chunks) PM Knowledge: complete — 23 indexed, 0 unchanged.

8.4 Subsequent Runs

On subsequent startups, Lliam checks file modification times and only re-indexes files that have changed. Startup remains fast even with large libraries. To force a full re-index, set reindexOnStart: true in your config.

8.5 Verify the Index

List all indexed sources in the knowledge base

Returns a table of every indexed file with chunk counts and timestamps.

9. Voice Channel & Phone Integration

9.1 Direct WebSocket (Microphone)

Any WebSocket client that streams PCM16 audio at 16 kHz can connect directly to ws://127.0.0.1:3001. Gemini Live handles turn-taking, barge-in, and function calling back into Lliam's tools.

9.2 Twilio Phone Integration

  1. Start an ngrok tunnel: ngrok http 3002
  2. Set the environment variable: export TWILIO_PUBLIC_URL=https://abc123.ngrok.io
  3. Configure your Twilio phone number's voice webhook: POST https://abc123.ngrok.io/twilio/voice
  4. Start Lliam. The Twilio adapter will serve TwiML at POST /twilio/voice and accept the Media Stream WebSocket at /twilio/media
  5. Call your Twilio number. Lliam transcodes mulaw 8 kHz audio to PCM16 16 kHz for Gemini, and converts responses back to mulaw for delivery to the caller

9.3 Phone Number Allowlist

To restrict who can call, set allowedPhoneNumbers in the gemini-voice plugin config:

"allowedPhoneNumbers": ["+15551234567", "+15559876543"]

Calls from unlisted numbers are rejected with a 4003 close code.

10. Security & Compliance

10.1 Encryption at Rest

All local databases — memory, knowledge base, and sessions — are encrypted with AES-256-GCM. The encryption key is generated on first boot, stored in your OS keychain (macOS Keychain or Linux libsecret), and never written to disk in plaintext.

10.2 Network Security

The gateway binds to 127.0.0.1 only. No service is accessible from outside your machine unless you explicitly proxy it — for example, via ngrok for Twilio integration.

10.3 Plugin Sandboxing

User-installed plugins from ~/.lliam/plugins/ run in isolated V8 heaps via isolated-vm:

  • Configurable memory limits (default 128 MB per plugin)
  • No access to Node.js APIs, the filesystem, or the network
  • Timeout enforcement: 5 seconds for registration, 30 seconds per tool call
  • Bridge-only communication through the pluginAPI interface

Bundled plugins (in plugins/core/ and plugins/executive/) are trusted and run in the main process.

11. Writing Custom Plugins

11.1 Plugin Structure

Create a directory in ~/.lliam/plugins/ with two files:

~/.lliam/plugins/my-plugin/ lliam.plugin.json # manifest index.ts # implementation

11.2 Manifest (lliam.plugin.json)

{ "id": "custom.my-plugin", "name": "My Plugin", "version": "1.0.0", "description": "What my plugin does", "kind": "general", "dependencies": [], "sandbox": { "enabled": true, "memoryLimitMB": 128 }, "configSchema": { "type": "object", "properties": {} } }

11.3 Plugin Implementation (index.ts)

import type { PluginModule, PluginAPI } from "path/to/plugin/types"; const myPlugin: PluginModule = { id: "custom.my-plugin", name: "My Plugin", version: "1.0.0", register(api: PluginAPI) { api.registerTool({ name: "my_tool", description: "What this tool does", parameters: { type: "object", properties: { input: { type: "string", description: "Tool input" }, }, required: ["input"], }, async execute(_toolCallId, params) { return { content: "Processed: " + String(params.input) }; }, }); api.logger.info("My plugin registered"); }, }; export default myPlugin;

12. Troubleshooting

"ANTHROPIC_API_KEY not set"

Export the environment variable or add it to a .env file in the project root. Lliam uses dotenv and will pick it up automatically on the next start.

"Failed to initialize Transformers.js embedding provider"

@xenova/transformers is not installed. It is an optional dependency — the PMI ontology and document generation tools work without it. Only knowledge base semantic search requires it:

npm install @xenova/transformers

"No documents indexed"

Verify that knowledgeBasePath is set in your plugin config and points to a valid directory containing .pdf, .md, or .txt files.

"PDF parsing requires pdf-parse"

npm install pdf-parse

Plugin fails to load

Check the diagnostic output at startup. Common causes:

  • Invalid lliam.plugin.json — must pass Zod schema validation
  • Module ID in code doesn't match the ID in the manifest
  • Missing entry point (index.ts or index.js)
  • Unresolved dependency — listed in the dependencies array but not found

Voice channel not starting

Verify GEMINI_API_KEY is set. Check that port 3001 is not in use:

lsof -i :3001

Twilio calls not connecting

Verify TWILIO_PUBLIC_URL is set and your ngrok tunnel is running. Check that the Twilio phone number's voice webhook points to your-url/twilio/voice using the POST method.

Gateway won't start (port conflict)

lsof -i :3000

"KeyManager not initialized"

The OS keychain must be unlocked. On macOS, unlock your Keychain in Keychain Access. In a CI or Docker environment, configure a fallback key via environment variable.
